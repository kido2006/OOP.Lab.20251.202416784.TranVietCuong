Question: Is JAVA a Pass by Value or a Pass by Reference programming language?
Answer: 
Java is Pass by Value programming language
For primitive types (like int, double, boolean), Java copies the actual value.
For objects, Java copies the reference value (the "address" that points to the object), not the object itself.

Question: After the call of changeTitle(jungleDVD, cinderellaDVD.getTitle()) why is the title of the JungleDVD changed?
Answer:
After the call of changeTitle(jungleDVD, cinderellaDVD.getTitle()), the title of jungleDVD changes because the reference to the same object is copied into the method. It modifies the data inside that object, not the reference itself.
In short, the reference was copied, but both references point to the same object. Therefore, it is modifying the object through one reference changes it for the other as well.
